    • 真机:

      • 因为用来学习docker的所以就用的虚拟机，真机系统是win10，网段是192.168.8.1~255

    • 虚拟机:

      • 用的vm，网卡是以桥接的方式连接的，相当于局域网的一台机子来使用

      • 系统版本：CentOS 7.3.1611

      • 为了方便调试就把CenOS的ip改为静态的，省的重启机子的时候ip变了,有中文注释的地方就是改的

    • 制作我的app镜像跑下容器

      • 编译打包springboot工程并上传到/root/workspace/app/ip目录下

        • 这里有一个非常重要的点那就是我怎么指定我项目里的数据库url，刚开始一直纠结spring boot项目如何配置mysql地址，因为在运行mysql容器时如果没有指定端口映射，是随机映射的，并且如果我们在mysql的url中写localhost:映射端口的话，那么我们使用link连接这两个容器的作用也就没有了。终于在看了一些资料后突然醒悟了，使用–link之后，docker会在子容器（这里的spring boot容器）的/etc/hosts中将父容器（这里的mysql容器）与父容器的ip地址绑定，那么我们就可以docker_mysql :3306来访问数据库了。也就是把application.yml中数据库url改为：

          ​

      • 制作镜像

      • 以zhdr/ip为镜像，用--name IpServer这个命令启动的容器命名，用--link docker_mysql连接之前启动的docker_mysql容器

        docker run -p 8088:8080 -d --name IpServer --link docker_mysql zhdr/ip

    • 到里俩个镜像已经制作完成了，并以俩个镜像启动的容器也已经连接并跑起来
